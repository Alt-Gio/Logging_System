# Role Hierarchy & Invite-Only Authentication Guide

## 🎯 Overview

The DICT DTC Logbook now features a **role-based hierarchy system** with **invite-only authentication** powered by Clerk. This ensures only authorized staff can access the admin panel.

---

## 👑 Role Hierarchy

### **SuperAdmin** (Highest Authority)
- **Badge:** Purple gradient badge with crown icon 👑
- **Who gets it:** The **first account** created in the system
- **Permissions:**
  - Full system access
  - Can manage all roles
  - Can invite Admins and Staff
  - Can change all settings
  - Cannot be demoted

### **Admin** (Full Access)
- **Badge:** Blue badge
- **Who gets it:** Invited by SuperAdmin or other Admins
- **Permissions:**
  - Full admin panel access
  - Can invite other Admins and Staff
  - Can manage settings
  - Can view/edit logs, PCs, network
  - Can create announcements

### **Staff** (View-Only)
- **Badge:** Gray badge
- **Who gets it:** Invited by SuperAdmin or Admins
- **Permissions:**
  - View-only access to dashboard
  - Can view logs and statistics
  - Cannot edit settings
  - Cannot invite users
  - Cannot modify data

## 🔒 Invite-Only System

### How It Works

**No public signup** - Users can only create accounts if they:
1. Receive an **email invitation** from an existing Admin/SuperAdmin
2. Use a **one-time link** generated by an Admin/SuperAdmin

### Invitation Methods

#### **Method 1: Email Invitations**
1. Go to **Admin → Settings → Staff & Auth**
2. Enter email addresses (one per line, or comma/semicolon separated)
3. Click **"Send Invitations"**
4. Clerk sends invitation emails automatically
5. Recipients click the link and create their account
6. New accounts default to **Admin** role

**Example:**
```
staff1@dict.gov.ph
staff2@dict.gov.ph, staff3@dict.gov.ph
```

#### **Method 2: One-Time Links**
1. Go to **Admin → Settings → Staff & Auth**
2. Switch to **"🔗 One-time Link"** tab
3. Click **"Generate Link"**
4. Copy the generated link (valid for 7 days)
5. Share the link with the person you want to invite
6. They click it and create their account
7. Link expires after first use or 7 days

**Use cases:**
- Quick onboarding without knowing email in advance
- Temporary access for consultants
- Emergency access grants

## 📊 Invitation Management

### Dashboard Stats

The Staff & Auth tab shows:
- **⏳ Pending** - Invitations sent but not yet accepted
- **✅ Accepted** - Users who created accounts
- **📋 Total** - All invitations ever sent

### Invitation Status

Each invitation shows:
- **Email address** (if email invite)
- **Status:** Pending, Accepted, Expired, Revoked
- **Created date**
- **Invitation link** (for one-time links)
- **Actions:** Copy link, Revoke

### Revoking Invitations

To cancel a pending invitation:
1. Find the invitation in the list
2. Click **"Revoke"** button
3. The invitation link becomes invalid
4. User can no longer sign up with that invite

## 🚀 First-Time Setup

### Creating the SuperAdmin Account

**Step 1: Deploy to Railway**
- Push code to GitHub
- Railway auto-deploys
- No accounts exist yet

**Step 2: First Login**
1. Visit `https://dict-logbook.up.railway.app/admin`
2. Click **"Sign Up"** (only available when no accounts exist)
3. Create account with your email
4. **This account automatically becomes SuperAdmin** 👑

**Step 3: Lock Down Access**
1. After first account created, sign-up is disabled
2. Only invited users can create accounts
3. SuperAdmin can now invite Admins and Staff

## 🔄 Workflow Examples

### **Scenario 1: Onboarding New Admin**

1. SuperAdmin logs in
2. Goes to **Settings → Staff & Auth**
3. Enters new admin's email: `newadmin@dict.gov.ph`
4. Clicks **"Send Invitations"**
5. New admin receives email from Clerk
6. New admin clicks link, creates account
7. Account created with **Admin** role
8. New admin can now access admin panel

### **Scenario 2: Temporary Staff Access**

1. Admin generates one-time link
2. Shares link with temporary staff via messaging app
3. Staff clicks link, creates account
4. Account created with **Admin** role (can be changed later)
5. After project ends, Admin revokes access via Clerk

### **Scenario 3: Role Management**

Currently, roles are set at invitation time:
- All invites default to **Admin** role
- SuperAdmin is only the first account
- Future enhancement: Role selection during invitation

## 🛠️ Technical Implementation

### Database Schema

```prisma
model Admin {
  id          String   @id @default(cuid())
  username    String   @unique
  password    String
  name        String
  role        Role     @default(STAFF)
  lastLoginAt DateTime?
  logs        AdminLog[]
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

enum Role {
  SUPER_ADMIN
  ADMIN
  STAFF
}
```

### Clerk Configuration

**Environment Variables:**
```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY="pk_test_..."
CLERK_SECRET_KEY="sk_test_..."
```

**API Endpoints:**
- `/api/invitations` - List all invitations
- `/api/invitations` (POST) - Send email invitations
- `/api/invitations/link` (POST) - Generate one-time link
- `/api/invitations/[id]` (DELETE) - Revoke invitation

## 📝 Best Practices

### **For SuperAdmin:**
1. Create your account first (becomes SuperAdmin automatically)
2. Immediately invite at least one backup Admin
3. Use email invitations for permanent staff
4. Use one-time links for temporary access
5. Regularly review and revoke unused invitations

### **For Admins:**
1. Only invite trusted individuals
2. Verify email addresses before sending invites
3. Revoke invitations if person doesn't join within 7 days
4. Monitor invitation acceptance rate

### **Security Tips:**
1. Enable 2FA in Clerk settings (optional but recommended)
2. Use strong passwords
3. Don't share one-time links publicly
4. Revoke access immediately when staff leaves
5. Audit invitation logs regularly

## 🆘 Troubleshooting

### "I can't invite users"
- **Check:** Are you logged in as Admin or SuperAdmin?
- **Check:** Is Clerk properly configured with API keys?
- **Check:** Are email addresses valid?

### "Invitation link doesn't work"
- **Check:** Has the link expired? (7 days)
- **Check:** Was it already used? (one-time only)
- **Check:** Was it revoked?

### "I don't see my role badge"
- **Refresh** the page
- **Clear cache** and reload
- **Check** if you're logged in

### "How do I change someone's role?"
- Currently, roles are set at invitation time
- Future update will add role management
- For now, revoke and re-invite with desired role
